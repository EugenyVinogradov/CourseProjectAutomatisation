##  2.  Перечень используемых инструментов с обоснованием выбора
### Для автоматизации будет использоваться JAVA версии 11, IDE Intellij IDEA с системой сборки Gradle, с подключенными зависимостями:
* lombok - чтобы исключить написание шаблонного кода для генерации геттеров, сеттеров, конструкторов и некоторых методов
* junit-jupiter - для подключения тестовой среды в IDE, использования аннотаций и тестовых методов. Использовать будем JUnit5, он более гибкий, чем JUnit4, и в то же время позволяет запускать тесты JUnit4
* selenide - будем использовать для взаимодействия с браузером, selenide по сравнению с selenium позволяет писать более лаконичный код, позволяет не следить за соответствием версии браузера и веб-драйвера, помогает решить вопросы со стабильностью тестов, решая проблемы Ajax и Timeout
* rest-assured - будем использовать для тестирования запросов через API
* DbUtils - будем использовать для взаимодействия с БД приложения
* GSON - будем использовать для конвертации JSON-объектов в Java-объекты и конвертации Java-объектов в JSON-объекты
* Faker - будем использовать для генерации тестовых данных
* Allure - будем использовать для автоматической генерации отчетов

### Также потребуется следующее ПО и ресурсы:
* Docker - для запуска контейнеров с БД
* Git - для контроля версий разработки тестов
* GitHub - для хранения проекта
